# Overlays

Each subdirectory here is a cluster overlay — the cluster-specific configuration
that is merged with the base templates from `../talos-bootstraper/base/` during rendering.

In production, each overlay is a **separate Git repository** on your private Gitea
instance, registered here as a git submodule. The `example/` directory is an inline
reference showing the expected structure.

## Creating a new overlay submodule

1. Create a new repo on Gitea (e.g. `mycluster-overlay`)

2. Copy an example from the submodule and push it:

```bash
cp -r ../talos-bootstraper/overlays/yourCluster-l2 /tmp/mycluster
mv /tmp/mycluster/yourCluster-l2.env /tmp/mycluster/mycluster.env
cd /tmp/mycluster && git init && git add -A && git commit -m "initial overlay"
git remote add origin https://gitea.example.com/my-org/mycluster-overlay.git
git push -u origin main && cd -
```

3. Add as a submodule:

```bash
git submodule add https://gitea.example.com/my-org/mycluster-overlay.git overlays/mycluster
```

4. Edit the `.env` file and set at minimum:
   - `OVERLAY_NAME` (must match the directory name)
   - `CLUSTER_EXTERNAL_DOMAIN`
   - `TALOS_CONTROL_NODES` and `TALOS_WORKER_NODES`
   - `CILIUM_LB_IP_CIDR`

Refer to `../talos-bootstraper/overlays/yourCluster-l2/yourCluster-l2.env` for a
fully documented example of every available variable.

## Overlay structure

```
mycluster/
├── mycluster.env          # All cluster configuration variables
├── talos/                 # Generated Talos machine configs + secrets
│   ├── controlplane.yaml
│   ├── worker.yaml
│   ├── talos-secrets.yaml
│   └── talosconfig
├── cilium/                # Optional network config overrides
│   └── ciliumHelmValues.yaml
└── _rendered/             # Auto-generated by render-overlay.sh (committed, ArgoCD reads from here)
    ├── argocd/
    ├── cilium/
    ├── gitea/
    └── ...
```

The `_rendered/` directory is populated by `render-overlay.sh` and **must be committed**
to the overlay repo — ArgoCD reads manifests from it directly.
